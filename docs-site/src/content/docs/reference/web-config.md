---
title: Web Dashboard Configuration
---

This document describes the configuration for the Scion Web Dashboard frontend and its Backend-for-Frontend (BFF).

## Purpose
The Web Dashboard is configured primarily through environment variables that control its connectivity to the Scion Hub, authentication flow, and session persistence.

## Core Environment Variables

| Variable | Default | Description |
| :--- | :--- | :--- |
| `PORT` | `8080` | The HTTP port the web server listens on. |
| `HOST` | `0.0.0.0` | The network address to bind to. |
| `NODE_ENV` | `development` | Set to `production` for optimized builds and strict security. |
| `HUB_API_URL` | `http://localhost:9810` | The endpoint of the Scion Hub API. |
| `BASE_URL` | (calculated) | The public URL of the dashboard, used for OAuth callbacks. |
| `SCION_API_DEBUG` | `false` | Enable verbose logging of requests and responses. |

## Authentication

### OAuth Providers
These variables are required for standard user login in production.

| Variable | Description |
| :--- | :--- |
| `GOOGLE_CLIENT_ID` | Google OAuth 2.0 Client ID. |
| `GOOGLE_CLIENT_SECRET` | Google OAuth 2.0 Client Secret. |
| `GITHUB_CLIENT_ID` | GitHub OAuth App Client ID. |
| `GITHUB_CLIENT_SECRET` | GitHub OAuth App Client Secret. |
| `SCION_AUTHORIZED_DOMAINS` | Comma-separated list of email domains allowed to sign in. |

### Development Authentication
Used for local testing without setting up full OAuth.

| Variable | Default | Description |
| :--- | :--- | :--- |
| `SCION_DEV_AUTH_ENABLED` | `true` (dev) | Enables the auto-login "Development User". |
| `SCION_DEV_TOKEN` | | Explicit development token for Hub API access. |
| `SCION_DEV_TOKEN_FILE` | `~/.scion/dev-token` | Path to the token file generated by the Hub. |

## Session Management
Required for persisting user logins across browser restarts.

| Variable | Default | Description |
| :--- | :--- | :--- |
| `SESSION_SECRET` | (random in dev) | Secret key for signing session cookies. **Must be set in production.** |
| `SESSION_MAX_AGE` | `86400000` | Session duration in milliseconds (default 24 hours). |

## Security Settings
The dashboard includes a pre-configured Content Security Policy (CSP) that allows connections to the Hub and necessary CDNs (Web Awesome, Shoelace).

- **CORS_ORIGIN**: Controls allowed origins for incoming requests (default `*`).
- **HSTS**: Automatically enabled in production with a 1-year max-age.

## Deployment
The Web Dashboard is designed to be deployed as a containerized service (e.g., on Cloud Run or Kubernetes). Ensure all required production variables (`GOOGLE_*`, `SESSION_SECRET`, `BASE_URL`) are provided in your deployment configuration.
